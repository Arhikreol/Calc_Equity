# Веб-страница на VPS для анализа Excel-отчетов

## Summary
- Сделать браузерную страницу, которая будет лежать на VPS с Windows Server 2022 и открываться по адресу вида `http://<IP_вашего_VPS>/equity-viewer/`.
- Для v1 взять самый простой и надежный вариант: вся логика работает на стороне браузера. Пользователь загружает свой `.xlsx`, браузер сам парсит файл, строит последовательность сделок и сразу пересчитывает график при движении ползунков.
- Файлы и настройки хранятся только в памяти конкретной вкладки браузера. Это автоматически дает изоляцию: один пользователь не увидит чужой файл и не повлияет на чужие настройки.

## Key Changes
- Вместо текущего desktop-варианта на `matplotlib` сделать веб-интерфейс с:
  - загрузкой `.xlsx`
  - ползунком `reset-after-losses` (`2..10`, шаг `1`)
  - ползунком множителя мартингейла (`1.0..10.0`, шаг `0.1`)
  - полем лимита максимальной просадки для автоподбора
  - графиком equity
  - правой панелью статистики: ratio, итоговый PnL, max drawdown, peak stake, число сделок, число плюсовых, число минусовых, winrate, max loss streak
- Разбор Excel делать прямо в браузере через библиотеку типа SheetJS:
  - брать первый лист
  - сначала сортировать строки по колонке `E` по возрастанию, от старых к новым
  - затем формировать последовательность по текущему правилу:
    - `J > 0` и `J != I` => `+1`
    - `J < 0` => `-1`
    - все остальные строки пропускать
- Математику мартингейла повторить в JavaScript один в один с текущим Python-инструментом:
  - базовая ставка фиксирована `$1`
  - payout фиксирован `92%`
  - после выигрыша следующая ставка сбрасывается на базовую
  - после `N` подряд минусов следующая ставка тоже сбрасывается на базовую и серия мартингейла начинается заново
- Развернуть как статический сайт на VPS:
  - рекомендуемый v1-вариант для Windows Server 2022: IIS со статической страницей
  - путь страницы: `/equity-viewer/`
  - без базы данных
  - без серверного хранения файлов
- Публичный интерфейс v1:
  - одна страница `index.html`
  - один клиентский JS-бандл для загрузки файла, парсинга, симуляции, автоподбора и отрисовки графика
  - без backend API в v1

## Test Plan
- Проверка парсинга файла:
  - пример Pocket Option export корректно читается с первого листа
  - сортировка по колонке `E` реально переставляет строки в порядок от старых к новым до формирования последовательности
  - строки с `J == I`, `J == 0` или пустыми `I/J` пропускаются
- Проверка симуляции:
  - известная тестовая последовательность воспроизводит текущее поведение `reset-after-losses`
  - автоподбор учитывает лимит `max drawdown`
  - если ни один конфиг не проходит по лимиту, интерфейс показывает понятную ошибку
  - статистика и график обновляются после загрузки файла и после движения ползунков
- Проверка изоляции пользователей:
  - открыть страницу в двух разных браузерах или в обычном окне и инкогнито
  - загрузить разные файлы
  - убедиться, что графики, статистика и настройки не влияют друг на друга
- Проверка деплоя:
  - страница открывается по IP VPS и нужному пути
  - крупные `.xlsx` не кладут интерфейс
  - обновление страницы сбрасывает состояние, если отдельно не внедрять сохранение

## Assumptions
- В v1 поддерживаются только `.xlsx` файлы формата, совместимого с `export_history_90147176776fe92d223ded6eef57f6ec.xlsx`.
- Источник данных это первый лист, заголовки находятся в первой строке, нужные колонки это `E`, `I`, `J`.
- Изоляция пользователей в v1 означает изоляцию на уровне браузера, без аккаунтов и авторизации.
- Постоянное хранение загруженных файлов на VPS не требуется.
- Для v1 допустим доступ по IP и пути страницы без отдельного домена и без HTTPS; это можно добавить позже.
